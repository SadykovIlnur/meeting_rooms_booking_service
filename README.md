# MEETING_ROOMS_BOOKING_SERVICE


## Описание проекта

«Сервис бронирования переговорных комнат» это веб-сервис для автоматизации бронирования переговорных комнат в коворкинге. В системе есть несколько заранее заданных комнат, несколько заранее заданные временных слотов. Помимо комнат в системе есть сотрудники и администраторы. Сотрудники могут просматривать доступность всех комнат на конкретную дату, создавать и отменять свои бронирования (доступ к чужим броням запрещён). Администраторы, дополнительно к возможностям сотрудников, могут отменять любые бронирования. 
API для проекта MEETING_ROOMS_BOOKING_SERVICE. Через этот интерфейс могут работать мобильные приложения или чат-бот; данные через этот API можно передавать и на фронтенд.
Проект полностью готов для развертывания на сервере в сети из 2 Docker-контейнеров: контейнере бэкенда, контейнере БД.


## Данные, необходимые для работы с проектом:

- Переменные окружения; указываются в файле ```.env```, расположенном в корне проекта. Пример заполнения указан в файле .env.example.
```bash
DB_USER — имя пользователя БД.
DB_PASSWORD — пароль пользователя БД.
DB_NAME — имя БД.
DB_HOST — адрес, по которому приложение будет соединяться с базой данных. При работе в сети Docker network вместо адреса указывают имя контейнера, где запущен сервер БД, — в нашем случае это контейнер db.
DB_PORT — порт, по которому Django будет обращаться к базе данных. 5432 — это порт по умолчанию для PostgreSQL.
DB_ECHO - необходимость логирования SQL-запросов (True/False).
SECRET_KEY — "секретный ключ" для подписи JWT.
JWT_ALGORITHM - алгоритм шифрования токена (по умолчанию HS256).
ACCESS_TOKEN_EXPIRE_DAYS - срок действия токена в днях.
```


## Данные, необходимые для работы с API

Возможности сервиса доступны только аутентифицированным пользователям.
Аутентификация осуществляется с помощью JWT-токена. Для получения токена аутентификации необходимо отправить POST-запрос на адрес /api/v1/auth/login, передать в теле запроса логин и пароль пользователя.


## Документация для API:

После запуска проекта по адресу (http://localhost:8000/docs) и (http://localhost:8000/redoc) будет доступна документация для API. В документации описано, как работает API: приведены примеры запросов и ожидаемых ответов, коды ошибок.


## Доступные эндпоинты API:

- health (GET): проверка работоспособности приложения (Ответ: {"status": "ok"}).
- api/v1/auth/login (POST): передача логина и пароля, получение JWT-токена.
- api/v1/slots (GET): получение списка всех временных слотов.
- api/v1/rooms (GET): получение списка всех комнат.
- api/v1/rooms/availability?date=YYYY-MM-DD (GET): получение списка доступных слотов комнат с флагом занятости для каждой комнаты.
- api/v1/bookings (POST): бронирование комнаты на временной слот.
- api/v1/bookings/my (GET): получение списка бронирований пользователя.
- api/v1/bookings/{booking_id} (DELETE): отмена бронирования (своего или любого для админа).
- api/v1/admin/bookings (GET): получение списка всех бронирований пользователей (только для админа).


## Пользовательские роли и права доступа

__Возможности неавторизованных пользователей:__
- Получение токена.

__Возможности авторизованных пользователей:__
- Просмотр списка комнат.
- Просмотр списка слотов.
- Просмотр доступности слотов на конкретную дату.
- Создание и удаление собственных броней.
- Просмотр собственных броней.

__Возможности администратора:__
- Все права авторизованного пользователя.
- Удаление любых броней.
- Просмотр всех броней с возможностью фильтрации по пользователю, комнате, дате.


## Запуск проекта на сервере

1. Клонировать репозиторий 
```bash
git clone git@github.com:SadykovIlnur/meeting_rooms_booking_service.git
```
2. Создать и заполнить файл .env
3. Запустить контейнеры
```bash
sudo docker compose up -d
```
Применение миграций и загрузка начальных данных происходят автоматически при старте приложения.
При запуске приложения автоматически:
1. заполняется таблица rooms тремя комнатами: «Комната 1», «Комната 2», «Комната 3» из CSV-файла rooms.csv, расположенного в app/data/.
2. заполняется таблица timeslots временными слотами:
  - 09:00–11:00
  - 11:00–13:00
  - 13:00–15:00
  - 15:00–17:00
  - 17:00–19:00
из CSV-файла timeslots.csv, расположенного в app/data/.
3. заполняется таблица users:
  - employee1 / employee123 (роль employee)
  - employee2 / employee456 (роль employee)
  - employee3 / employee789 (роль employee)
  - admin / admin123 (роль admin)
из CSV-файла users.csv, расположенного в app/data/.


## Примеры запросов

1. Пример POST-запроса: получение токена авторизации.
POST .../api/v1/auth/login
```bash
{
    "username": "employee1",
    "password": "employee123"
}
```
Пример ответа:
```bash
{
  "access_token": "string",
  "token_type": "bearer"
}
```

2. Пример GET-запроса: получение списка всех комнат.
GET .../api/v1/rooms
Пример ответа:
```bash
[
  {
    "id": 1,
    "name": "Комната 1"
  },
  {
    "id": 2,
    "name": "Комната 2"
  },
  {
    "id": 3,
    "name": "Комната 3"
  }
]
```

3. Пример POST-запроса: бронирование комнаты на временной слот.
POST .../api/v1/bookings
```bash
{
    "room_id": 1,
    "timeslot_id": 1,
    "booking_date": "2026-06-10"
}
```
Пример ответа:
```bash
{
  "room_id": 1,
  "timeslot_id": 1,
  "booking_date": "2026-06-10",
  "id": 1
}
```

4. Пример GET-запроса: получение списка доступных слотов комнат с флагом занятости для каждой комнаты.
GET .../api/v1/rooms/availability?date_str=2026-06-10
Пример ответа:
```bash
[
  {
    "room_id": 1,
    "room_name": "Комната 1",
    "slots": [
      {
        "slot_id": 1,
        "start_time": "09:00:00",
        "end_time": "11:00:00",
        "is_booked": true
      },
      {
        "slot_id": 2,
        "start_time": "11:00:00",
        "end_time": "13:00:00",
        "is_booked": false
      },
      {
        "slot_id": 3,
        "start_time": "13:00:00",
        "end_time": "15:00:00",
        "is_booked": false
      },
      ...
    ]
  }
]
```

5. Пример GET-запроса: получение списка всех бронирований пользователей (только для админа).
GET .../api/v1/admin/bookings?user_id=1&booking_date=2026-06-10&room_id=1
Пример ответа:
```bash
[
  {
    "id": 1,
    "booking_date": "2026-06-10",
    "room": {
      "id": 1,
      "name": "Комната 1"
    },
    "timeslot": {
      "id": 1,
      "start_time": "09:00:00",
      "end_time": "11:00:00"
    },
    "user": {
      "id": 1,
      "username": "employee1",
      "role": "employee"
    }
  }
]
```
